# Ink TUI 交互设计

## 目标

`chat-codex` 的 TUI 是现有核心交互的展示层，不是第二套业务逻辑。

目标体验：

- 首页一眼看懂渠道、聊天绑定、权限和是否可以启动。
- 首页展示新 session 默认工作目录，避免在错误仓库里创建 Codex session。
- 所有对象列表支持方向键、数字编号和快捷键。
- 常见动作不需要记命令：添加微信、添加飞书、绑定 session、设置权限、启动服务。
- 成功、失败和下一步显示在固定状态栏，用户不需要翻终端历史。
- TUI 不直接读写本地状态文件，不直接判断 session owner 冲突，不直接 new 具体渠道 adapter 做业务判断。

非目标：

- 不做网页式复杂布局。
- 不改变微信一个账号一个主聊天、飞书一个机器人多个私聊 `chat_id` 的产品模型。
- 不改变 session 唯一归属规则。

## 技术选择

第一版使用 Ink。

原因：

- 项目当前是 Node.js + TypeScript，Ink 可以直接进入现有构建链路。
- React 组件模型适合 dashboard、列表、详情、底部状态栏和弹窗。
- `useInput` 可以处理方向键、回车、Esc 和快捷键。
- Ink 的测试工具可以对组件输出做单元测试。

可选配套：

- `@inkjs/ui`：用于输入框、密码框、确认框、选择框等基础控件。
- `ink-testing-library`：用于 TUI 组件测试。

暂不采用：

- `blessed` / `react-blessed`：能力强但生态偏旧，React 版本约束和维护风险更高。
- `terminal-kit`：控制力强，但更偏命令式，和后续组件测试不如 Ink 顺。
- `OpenTUI`：能力很强，但当前 Node 支持还不适合作为本项目主线。

## 分层边界

```text
Ink TUI Components
        ↓
LauncherActions / ChannelActions / BindingActions
        ↓
State Store / Channel Config Store / Codex Session Discovery
        ↓
Bridge Runtime
```

TUI 组件允许：

- 展示 actions 返回的结构化状态。
- 处理键盘输入。
- 调用 actions。
- 显示加载中、成功、错误和下一步提示。

TUI 组件禁止：

- 直接写 `state/bridge/*.json`。
- 直接读写 `session-owners.json` 判断 owner 冲突。
- 直接 new `WeixinAdapter` 或 `FeishuAdapter` 做业务判断。
- 直接调用 Codex adapter 创建 session。
- 在组件里实现微信 pending binding 或飞书多 `chat_id` 绑定规则。

## 入口行为

推荐入口保持不变：

```bash
npm run chat-codex
```

默认行为：

- TTY 环境：进入 Ink TUI。
- 非 TTY 环境：走现有 prompt fallback 或直接报错提示需要交互。
- `--no-tui`：强制走普通 prompt 交互。
- `--no-interactive`：不进入交互，只按现有非交互规则检查并启动。

`npm run cli:chat-codex` 保持同等别名。

长期分工：

- TTY 默认使用 TUI，后续正常人工启动都优先走 TUI。
- 普通 prompt CLI 必须保留，用于 `--no-tui`、非 TTY、自动化脚本、远程日志环境和 TUI 故障排查。
- TUI 和 prompt CLI 必须调用同一套 actions/services，不能分叉出两套业务逻辑。

- 不要求第一版必须实现左右栏。

## 全局键盘

全局键：

- `Up` / `Down`：移动当前选中项。
- `Enter`：执行当前主操作，或进入当前选中项。
- `Esc`：返回上一级。
- `q`：返回上一级；在首页退出。
- `r`：刷新当前页面数据。
- `?`：显示当前页面快捷键说明。

首页快捷键：

- `Enter`：启动服务。如果当前不能启动，进入需要处理的页面。
- `c`：管理渠道。
- `b`：聊天绑定。
- `p`：默认权限设置。
- `d`：新 session 默认工作目录。
- `s`：状态详情。
- `q`：退出。

渠道页快捷键：

- `w`：添加微信账号。
- `f`：添加飞书机器人。
- `Enter`：进入选中渠道详情。
- `e`：启用或停用选中渠道。
- `Esc` / `q`：返回首页。

绑定页快捷键：

- `Enter`：进入选中聊天绑定详情。
- `n`：给选中聊天新建并绑定 session。
- `m`：手动输入 session ID 绑定选中聊天。
- `u`：解绑选中聊天当前 session，需二次确认。
- `p`：设置选中聊天当前 session 权限。
- `Esc` / `q`：返回首页。

Session 选择页快捷键：

- `Enter`：绑定当前选中的可选 session。
- 数字：直接选择对应 session。
- `n`：新建 session。
- `m`：手动输入 session ID。
- `Esc` / `q`：返回。

## 页面状态

每个页面统一维护这些 UI 状态：

- `idle`：正常展示。
- `loading`：正在刷新、检查状态、创建 session 或启动服务。
- `input`：正在输入文本或 secret。
- `confirm`：等待用户二次确认。
- `error`：操作失败，底部显示错误，仍停留当前页面。
- `success`：操作成功，底部显示结果和下一步。

底部状态栏规则：

- 成功：显示绿色短句，例如 `已绑定 session：你好呀 / 019e3024`。
- 错误：显示红色短句，例如 `无法绑定：该 session 已绑定到 飞书 / default / 张三`。
- 提示：显示灰色或普通文本，例如 `启动服务后，在飞书里私聊机器人会自动记录 chat_id`。
- 密钥、token、secret 不显示在状态栏。

## 首页

首页展示五块：

- 渠道：每个渠道实例的类型、账号、连接状态和关键下一步。
- 聊天绑定：微信主聊天状态、飞书已发现私聊数量、未绑定数量。
- 权限：新 session 默认权限。
- 工作目录：新 session 默认工作目录，长路径省略，详情页完整展示。
- 运行提示：是否可以启动，不能启动时显示需要处理的第一件事。

无渠道时：

```text
┌ Chat Codex ─────────────────────────────────────────────┐
│ 状态: 未配置                                  权限: 审批 │
├─────────────────────────────────────────────────────────┤
│ 渠道                                                     │
│  暂无渠道                                                │
│                                                         │
│ 下一步                                                   │
│  按 w 添加微信账号，或按 f 添加飞书机器人。               │
├─────────────────────────────────────────────────────────┤
│ w 添加微信   f 添加飞书   p 权限   q 退出                 │
└─────────────────────────────────────────────────────────┘
```

启动规则：

- 有至少一个 enabled 且 connected 的渠道时，`Enter` 进入启动确认。
- 没有渠道时，`Enter` 进入管理渠道。
- 有渠道但不可启动时，`Enter` 跳到第一个需要处理的渠道详情。

## 添加微信账号

流程：

1. TUI 调用 channel setup action 发起扫码登录。
2. 页面显示二维码和备用链接。
3. 用户扫码并手机确认。
4. 按 `Enter` 检查登录结果。
5. 按 `Esc` / `q` 返回管理渠道，不添加账号。
6. 登录成功后自动进入微信主聊天绑定页。

页面：

```text
┌ 添加微信账号 ───────────────────────────────────────────┐
│ 请使用微信扫码，并在手机上确认。                         │
│                                                         │
│  <二维码区域>                                            │
│                                                         │
│ 状态: 等待确认                                           │
├─────────────────────────────────────────────────────────┤
│ Enter 检查登录结果   Esc 返回                            │
└─────────────────────────────────────────────────────────┘
```

登录结果：

- 成功：注册微信渠道实例，底部显示账号和下一步。
- 超时：停留当前页，提示继续按 `Enter` 检查或返回。
- 失败：停留当前页或返回渠道页，显示失败原因。

## 添加飞书机器人

流程：

1. TUI 直接进入手动输入表单。
2. 输入 App ID。
3. 输入 App Secret，必须用密码框隐藏。
4. 输入 domain，默认 `feishu`。
5. 输入账号标识，默认 `default`。
6. 调用 action 检查机器人身份。
7. 成功后注册飞书渠道实例。

规则：

- App Secret 只保存在当前进程内存里。
- 不写入 state 文件。
- 不显示在页面、状态栏、错误文案和日志里。
- 重启后需要环境变量或重新手动添加。
- 添加成功后不进入 session 绑定。

成功页提示：

```text
飞书机器人已添加。
启动服务后，让用户在飞书里私聊机器人。
每个飞书私聊会按 chat_id 生成独立聊天绑定。
```

## 微信主聊天绑定

微信主聊天绑定复用 Session 选择组件，但目标不是普通 routeKey，而是微信主聊天 target。

状态：

- 已发现真实微信私聊 route：直接绑定该 route。
- 未发现真实微信私聊 route：写 pending binding。

操作：

- 选择已有 session：pending 预留该 session，收到第一条微信私聊后生效。
- `n` 新建：收到第一条微信私聊后创建新 session。
- `0` / `Esc`：暂不绑定，首条消息按新聊天策略处理。

显示：

```text
待绑定到 你好呀 / 019e3024
说明: 收到第一条微信私聊后生效。
```


- TUI 不展示 Codex adapter 模式、progress 模式、并发上限等高级项。

```text
┌ 工作目录设置 ───────────────────────────────────────────┐
│ 当前                                                     │
│  /Users/xiaohuang/codex-wechat/codex-openclaw-wechat     │
│                                                         │
│ > 使用当前终端目录                                      │

└─────────────────────────────────────────────────────────┘
```

规则：

- 默认工作目录只影响以后新建的 session。

- 输入不存在的目录时，先显示确认提示，再创建目录并保存。
- 保存后刷新首页、启动确认页和运行日志页中的工作目录。
- 工作目录应通过 actions/services 持久化到本地配置，TUI 不直接写 `state/bridge/config.json`。

## 状态详情

状态详情用于排查，不作为主路径。

展示：

- 渠道：类型、账号、状态、启用状态、最近错误。
- 绑定：routes 数量、已绑定数量、pending 数量。
- 权限：默认权限、session 级权限数量。
- 运行：服务未启动或运行中。

状态详情可以显示完整 ID，但仍避免输出原始 JSON。

第一版启动后行为：

- TUI 退出配置界面后启动 Bridge。
- TTY 默认进入运行期日志面板，展示已启动渠道、工作目录、默认权限、新聊天策略和最近 transcript。

- `q` / `Esc` 停止服务并退出。

- 运行期 TUI 可以继续增强为左右两栏：左侧显示渠道和任务状态，右侧显示最近 transcript。
- 后续补充滚动、过滤、暂停跟随、审批状态和队列状态。

## 错误处理

通用规则：

- 用户输入错误不退出程序。
- 操作失败不丢失当前页面上下文。
- 错误文案使用中文可行动提示。
- 业务错误由 actions 返回结构化结果，TUI 只负责展示。

典型错误：

- 微信登录超时：提示继续检查或返回。
- 飞书凭证缺失：提示重新输入，不显示 secret。
- session 不存在：停留 Session 选择页。
- session owner 冲突：显示已绑定到哪个聊天。
- 无 enabled 渠道：跳转管理渠道。
- 有 enabled 渠道但未 connected：跳转对应渠道详情。

- 微信 pending 主聊天绑定。
- 飞书添加机器人不落盘 secret。
- session 选择 owner 冲突。

TUI 组件：

- 首页无渠道、可启动、不可启动三种状态快照。
- 首页和启动确认页展示工作目录。
- 渠道列表快捷键。
- 绑定列表和详情展示。
- Session 选择可选/不可选展示。
- 工作目录页输入、确认创建和返回行为。
- 错误状态栏展示。
- Esc/q 返回行为。

人工验证：

```bash
npm run chat-codex
```

重点验证：

- 窄终端和宽终端都不出现重叠。
- 中文宽字符对齐可接受。
- App Secret 输入不回显。
- 启动服务后 TTY 默认进入运行期日志面板，收到/发送/进度日志能追加显示，并展示当前工作目录。
- `--no-tui` 下 transcript 正常打印。
- 非 TTY fallback 不受影响。
